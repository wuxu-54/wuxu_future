# 声明变量关键字

## var的问题

var 容易引起bug

1. 不受作用域限制，声明后，全局可以调用
2. 相同变量无论声明几次都可以使用，这就容易引起bug

let 和 const 用于取代var

## let和const

const 是不可修改的变量。其他与let相同

1. 同名变量只能声明一次
2. 受到作用域限制，块作用域声明了变量，外部使用编译器会报错

## 解构

和js一模一样，看js章节。目前ts编译器不支持泛型函数解构。

补充，由于TS是强类型语言，解构在js基础上也可以写明结构属性对应的数据类型

```js
    let o = {a:"a",b:1,c:"bar"}
    //结构
    let {a,b} = o;//可以完全使用js方式
    let {a:newNameA,b:newNameB} :{a:string,b:number} = o;//ts 特有，可以指明数据类型;newNameA 、newNameB是声明为新名
```

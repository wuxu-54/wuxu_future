# 嵌套路由

顾名思义，路由允许嵌套配置，子路由等同于拼接在父路由路径后。

例如：

父路由path：/base
子路由path：child。此时完整的子路由path：/base/child

## 定义嵌套路由示例

下面例子中，DashboardLayout是父路由的组件，它可能包含一个导航菜单和主内容区域。DashboardOverview和DashboardSettings是子路由的组件，它们将被渲染在父组件的主要内容区域内。

```javascript
const routes = [
  {
    path: '/dashboard',
    component: DashboardLayout, // 父组件，通常包含布局
    children: [
      {
        path: '', // 子路由的默认路径
        component: DashboardOverview, // 子组件
      },
      {
        path: 'settings',
        component: DashboardSettings,
      },
      // 更多子路由...
    ],
  },
  // 其他路由...
];
```

路由视图：

```javascript
<template>
  <aside>
    <!-- 导航菜单 -->
    <router-link to="/dashboard">Overview</router-link>
    <router-link to="/dashboard/settings">Settings</router-link>
  </aside>
  <main>
    <!-- 子路由组件将在这里渲染 -->
    <router-view></router-view>
  </main>
</template>
```

上面视图示例除了使用`<router-link>`进行导航，你还可以使用编程式导航来切换子路由：

```javascript
import { useRouter } from 'vue-router';

export default {
  setup() {
    const router = useRouter();
    const goToSettings = () => {
      router.push('/dashboard/settings');
    };
    return {
      goToSettings,
    };
  },
};
```

## 嵌套命名路由

嵌套路由可以为子路由命名，显示规则与普通嵌套没有区别。

```javascript
const routes = [
  {
    path: '/user/:id',
    name: 'parent'
    component: User,
    // 请注意，子路由具有名称，但path为默认值，那么 /user/:id 导航时始终显示子路由。
    //如果想显示父路由,那么设定父路由name。请注意重新加载页面将始终显示嵌套的子路由
    children: [{ path: '', name: 'user', component: UserHome }],
  },
]
```
